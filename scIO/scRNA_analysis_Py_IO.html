reading data from Python scanpy¶

This data was generated through a tutorial based on Scanpy. We can see that adata.X is now storing scale data, and the adata.raw.X is now storing the log normal data.In general analysis, we use log data or counts data for downstream analysis.In the current version, our software does not advocate saving scale data.

The scanpy data is saved to the h5 file which is can be read in R. By default the adata.raw.X will be saved by set the save_raw=True. We do not recommend saving the scale data. If the user must save it, then adata.X to store scaleData and adata.raw.X to store log data or counts data with save_raw=None.Note that the dimensions need to be the same.
