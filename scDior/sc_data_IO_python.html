2. reading data from Python scanpy and save the h5 file¶

2.1 read data from h5ad file¶

This data was generated through a tutorial based on Scanpy tutorial. There are log normal data in adata.raw.X, cell annotation in adata.obs, all gene annotation in adata.raw.var, dimension reduction in adata.obsm, the cell-cell similarity network in adata.obsp, scale data in adata.X. meta data in adata.uns.In general analysis, we use log data or counts data for downstream analysis.In the current version, our sofware can save log normal data, counts, cell annotation, all gene annotation, dimension reduction, the cell-cell similarity network and some meta data exactly.
Our software does not save scale data and cell-cell similarity matrix by default.

data type Scanpy Note
coutnts matrix adata.X the raw data
log-normal data adata.raw.X
scale data adata.X not saved
cell annotation adata.obs
gene annotation adata.var
dimension reduction data.obsm
cell-cell similarty matrix adata.uns not saved

2.2 save data into h5 file¶

The scanpy data is saved to the h5 file which is can be read in R.Our software does not recommend saving scale data and gene expression matrix containing only HVGs.By default the adata.raw.X will be saved by set the save_raw=True

In [13]:
# save the data
myh5.output.write_h5(adata = adata, file = './py_scanpy.h5')

If the our target adata is in adata.X, the adata.X will be saved by set the save_raw=False

In [14]:
ad = anndata.AnnData(X=adata.raw.X, obs=adata.obs,var=adata.raw.var, obsm=adata.obsm)
In [15]:
# save the data 
myh5.output.write_h5(adata = ad, file='./py_scanpy_v1.h5', save_raw=False)
